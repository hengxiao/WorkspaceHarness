# Context Library

Searchable knowledge for everything an agent (or human) needs to know about the projects in this harness. Every document here carries YAML frontmatter (`title`, `tags`, `summary`, `updated`, `source`).

## Layout

| Directory | What goes here | Editable? |
| --- | --- | --- |
| `docs/` | Public-facing documentation written in this harness | yes |
| `internal/` | Internal notes, runbooks, gotchas, ADRs | yes |
| `specs/design/` | Design specs (architecture, components) | yes |
| `specs/requirements/` | Requirement specs (what we're building and why) | yes |
| `specs/implementation/` | Implementation-level specs and notes | yes |
| `bugs/` | Bug reports, postmortems, incident notes | yes |
| `upstream/<project>/` | DERIVED snapshots of submodule docs | **no** — regenerated by `make reindex` |

## Adding a document

1. Use `harness ctx add <path>` to scaffold frontmatter, **or** create the file by hand and include the standard frontmatter block.
2. Run `make reindex` to update `index.json`.
3. Commit.

## Searching

```
harness ctx search "<query>"
```

Default backend is SQLite FTS5 (see `.spec/design.md` §6.1). Opt-in semantic backends (Lance, Qdrant) are configured via `harness.yml`.
